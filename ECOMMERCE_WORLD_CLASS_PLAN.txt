PROFESSIONAL & MODERN E-COMMERCE PLATFORM

Goal
----
Build a production-ready, feature-rich e-commerce platform with gorgeous UI, professional design,
and complete admin control. Simple tech stack, complex features, beautiful experience.

Tech Stack (SIMPLE & PROVEN)
----------------------------
- Frontend: Next.js (App Router), React 18+, TypeScript, Tailwind CSS, shadcn/ui, Framer Motion
- Backend: Node.js + Express + TypeScript
- Database: PostgreSQL + simple connection pooling
- Auth: JWT tokens (no Redis needed)
- Payments: Stripe (test mode works fine)
- Email: Resend (simple API)
- File Storage: S3-compatible (local dev) or AWS S3 (production)
- UI Components: shadcn/ui, Radix UI, Lucide Icons
- Animations: Framer Motion for smooth interactions
- Forms: React Hook Form + Zod validation
- State: Zustand for simple state management
- Deployment: Vercel (frontend) + Railway/Render (backend)

================================================================================
UI/UX PRINCIPLES - Gorgeous & Professional
================================================================================

Design System
- Color scheme: Modern, professional (primary color + neutrals)
- Typography: Clean, readable (Geist or Inter font)
- Spacing: Consistent 8px grid system
- Shadows: Subtle depth (no harsh shadows)
- Borders: Subtle gray for subtle separation
- Icons: Lucide icons (consistent, clean)

Components (shadcn/ui based)
- Buttons: Multiple states (default, hover, active, disabled, loading)
- Forms: Clean labels, error messages in red, success in green
- Cards: Subtle shadows, proper padding
- Modals: Dark overlay, centered, smooth animation
- Dropdowns: Smooth open/close animation
- Tables: Zebra striping, hover rows, sortable headers
- Alerts: Color-coded (green=success, red=error, yellow=warning, blue=info)

Mobile Experience
- Touch-friendly tap targets (minimum 44x44px)
- Full-screen drawers for menus
- Swipe gestures for navigation
- Readable font sizes (minimum 16px on mobile)
- Responsive images
- Collapsible sections to reduce scrolling

Animations & Interactions
- Smooth page transitions (200-300ms)
- Hover effects on interactive elements
- Loading states with spinners
- Success/error animations
- Scroll animations (fade in on scroll)
- Form validation feedback (real-time)

Accessibility
- WCAG 2.1 AA compliant
- Keyboard navigation support
- Screen reader friendly
- Color contrast ratios met
- Focus indicators visible
- Alt text on all images

Database (PostgreSQL Schema)
----------------------------
Tables:
  users (id, name, email, password_hash, created_at, updated_at)
  
  products (id, name, description, price, cost_price, discount_price,
            sku, category, stock, status, image_url, created_at)
  
  product_variants (id, product_id, name, color, size, stock, price_override)
  
  orders (id, user_id, status, total, shipping_address, billing_address,
          created_at, updated_at)
  
  order_items (id, order_id, product_id, quantity, price, variant_id)
  
  reviews (id, product_id, user_id, rating, text, created_at)
  
  wishlist (id, user_id, product_id, created_at)
  
  coupons (id, code, discount_type, discount_value, min_purchase,
           max_uses, valid_from, valid_to)
  
  inventory (id, product_id, quantity, low_stock_threshold)
  
  admin_users (id, email, role, permissions, created_at)
  
Indexes on: email, sku, category, user_id, product_id, created_at

DEPLOYMENT STACK
----------------
Frontend:
  - Vercel (Next.js hosting, automatic deployments from git)
  - CDN for images (Vercel's built-in)

Backend:
  - Railway.app or Render.com
  - PostgreSQL hosting (managed database)
  - Environment variables for secrets

CI/CD:
  - GitHub Actions for linting, tests on PR
  - Automatic deploy on merge to main
